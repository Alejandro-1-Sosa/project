Tarea 1: Birfucar el Repositorio

Nota: No olvidar cambiar el nombre de la rama del repositorio de master a main

2- Guarde la URL de su Repositorio Birfucardo en un bloc de notas para enviarla más tarde para su revisión


Tarea 2: Corregir el error tipográfico y fusionarlo con principal

1- Clona el repositorio bifurcado en el entorno de laboratorio

git clone <"el nombre del repositorio">

2- Crea una rama llamada bug-fix-typo

git branch "bug-fix-typo"

3- Cambie el pie de página en el archivo README.md principal de

2022 XYZ, Inc.

a

2023 XYZ, Inc.


4- Agregue el archivo con su solución y confírmelo con un mensaje significativo.

git add README.md

git config --global user.email "your-email@example.com"

git config --global user.name "YourName"

git commit -m "Fixed typographical error in README.md"


5- Empuje su solución a la bug-fix-typo rama. En este paso, necesitarás generar un token de acceso personal desde GitHub.com para usarlo como contraseña. Siga las instrucciones de la práctica de laboratorio Generar token de acceso personal de GitHub .

git push origin bug-fix-typo

Nombre de Usuario de GitHub: Alejandro-1-Sosa

Contraseña: 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

6- Cambie a la rama principal. Fusiona la bug-fix-typorama nuevamente con tu mainrama. Tome una captura de pantalla que muestre la rama actual y la operación de fusión exitosa con el archivo que ha cambiado. Guarde la captura de pantalla como merge_branches.pngo merge_branches.jpg.

git checkout main

git merge bug-fix-typo

Toma una captura de pantalla de la fusión exitosa y guárdala como merge_branches.png o merge_branches.jpg


Tarea 3: Revertir el error tipográfico y enviar una Solicitud de Extracción

1- Verifique el contenido de README.md en la rama principal. El archivo ahora debería leer:

2023 XYZ, Inc.

2- Crea una nueva rama llamada bug-fix-revert.

git branch bug-fix-revert

3- Revierta el cambio que implementó en la tarea anterior usando el git revert comando. El archivo ahora debería leer:

2022 XYZ, Inc.

git revert <"hash_del_commit">


4- Empuje la reversión a su repositorio en la bug-fix-revertrama. Asegúrese de utilizar el token de acceso personal que generó en Github para su cuenta como contraseña (y no su contraseña de git real), cuando se le solicite.

git push origin bug-fix-revert

Ve a la interfaz de GitHub, crea una nueva solicitud de extracción desde bug-fix-revert a main

Anota la URL de la solicitud de extracción en un bloc de notas

5- Vaya a la interfaz de usuario de GitHub. Cree una nueva solicitud de extracción desde la bug-fix-revertrama de su repositorio a la mainrama del repositorio original . Este PR se cerrará automáticamente. Anote la URL de este PR en un bloc de notas para enviarlo más tarde para su revisión por pares .


Tarea 4: Consulta el Estado de tus Sucursales

1- Navegue a la Branchessección dentro de la interfaz de usuario de GitHub en su página. Estará en el siguiente formato:

https://github.com//jbbmo-Introduction-to-Git-and-GitHub/branches

2- Dentro de esta sección, encontrará los nombres de las sucursales junto con su estado actual.

3- Tome nota de la URL de esta página en un bloc de notas para enviarla durante la revisión por pares.
